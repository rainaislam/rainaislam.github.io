Research Projects

Emotions in Software Engineering -- Detecting emotions expressed in software engineering communication channels (e.g., comments on pull requests, issue discussions, chats) in order to reduce toxic or incivil comments, undestand developer's emotion towards a specific aspect or feature or of the project, etc. Contribute to a more sustainable open source software eco-system. [ASE 2022,ICSE 2024a,ICSE 2024b]
 
Conversational Interfaces for Software Engineering -- We design and develop conversational interfaces for 1) end-users to program cyber-physical systems (e.g., UAVs); 2) software engineering novices to find software development information on the Web; 3) developers to improve the quality of bug reporting [ITiCSE 2024,Elsevier IST 2022]
 
Software Vulnerability Data Quality -- Patches to known software vulnerabilities, as maintained by the National Vulnerability Database are often noisy or incorrect. Our work aims to understand and improve the quality of this important data source. It also aims to create a corpus of explained real-work vulnerabilities that can be leveraged in education. [Real-World Vulnerabilities in CS Education]
